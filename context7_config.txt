TITLE: Implementing Debounced Search with FutureProvider and StateProvider (Dart/Flutter)
DESCRIPTION: This comprehensive example demonstrates a full 'search as we type' implementation. It uses a StateProvider for the search field, a FutureProvider to fetch questions from the StackExchange API (handling empty queries and encoding), and a Flutter UI to display results, including loading and error states. ```
import 'package:flutter/material.dart';
import 'package:flutter_riverpod/flutter_riverpod.dart';
import 'package:http/http.dart' as http;
import 'dart:convert';

final searchFieldProvider = StateProvider<String>((ref) => '');
final questionsProvider = FutureProvider<List>((ref) async {
  final client = http.Client();
  ref.onDispose(client.close);

  final search = ref.watch(searchFieldProvider);

  Uri uri;

  if (search.isEmpty) {
    uri = Uri.parse(

                  },
                ),
              AsyncError(:final error) => Center(child: Text('Error $error')),
              _ => const Center(child: CircularProgressIndicator()),
            },
          ),
        ],
      ),

TITLE: Creating a Reusable Debounce Extension for Riverpod Ref
DESCRIPTION: This code defines an extension method `debounce` on Riverpod's `Ref` object. This utility encapsulates the logic for both debouncing and cancellation, allowing developers to apply this pattern consistently across multiple providers. It takes a future-returning function and a duration, executing the function only after the specified duration has passed without further triggers, or aborting if the provider is disposed.
SOURCE: https://github.com/rrousselgit/riverpod/blob/master/website/docs/case_studies/cancel.mdx#_snippet_4

LANGUAGE: Dart
CODE:
```
import 'dart:async';

import 'package:riverpod/riverpod.dart';

// A custom exception to signal that a provider was disposed before an operation completed.
class ProviderDisposeException implements Exception {
  const ProviderDisposeException();
}

extension RefX on Ref {
  /// Debounces an asynchronous operation.
  /// The [create] function will only be executed after [duration] has passed
  /// without the provider being disposed.
  /// If the provider is disposed before the duration, the operation is aborted.
  Future<T> debounce<T>(
    Future<T> Function() create,
    {Duration duration = const Duration(milliseconds: 500),}
  ) async {
    final completer = Completer<void>();
    final timer = Timer(duration, () {
      if (!completer.isCompleted) {
        completer.complete();
      }
    });

    onDispose(() {
      timer.cancel();
      if (!completer.isCompleted) {
        completer.completeError(const ProviderDisposeException());
      }
    });

    try {
      await completer.future; // Wait for the timer to complete or for dispose
    } on ProviderDisposeException {
      // If the provider was disposed, rethrow to indicate cancellation
      rethrow;
    } catch (e) {
      // Handle other potential errors during the wait
      rethrow;
    }

    // If we reach here, it means the timer completed, so execute the original future
    return create();
  }
}
```

import 'package:flutter_riverpod/flutter_riverpod.dart';
import 'package:state_notifier/state_notifier.dart';

// A simple Todo model
class Todo {
  Todo({required this.id, required this.description, this.completed = false});

  final String id;
  final String description;
  final bool completed;

  Todo copyWith({String? id, String? description, bool? completed}) {
    return Todo(
      id: id ?? this.id,
      description: description ?? this.description,
      completed: completed ?? this.completed,
    );
  }
}

// The StateNotifier that holds and manages the list of todos
class TodoListNotifier extends StateNotifier<List<Todo>> {
  TodoListNotifier([List<Todo>? initialTodos]) : super(initialTodos ?? []);

  void addTodo(Todo todo) {
    state = [...state, todo];
  }

  void toggle(String id) {
    state = [
      for (final todo in state)
        if (todo.id == id)
          todo.copyWith(completed: !todo.completed)
        else
          todo,
    ];
  }

  void remove(String id) {
    state = state.where((todo) => todo.id != id).toList();
  }
}

// The StateNotifierProvider that exposes the TodoListNotifier
final todoListProvider = StateNotifierProvider<TodoListNotifier, List<Todo>>((ref) {
  return TodoListNotifier([
    Todo(id: '1', description: 'Learn Riverpod', completed: false),
    Todo(id: '2', description: 'Build a todo app', completed: true),
  ]);
});

TITLE: Cancelling Async Operations in Riverpod Notifiers without `mounted`
DESCRIPTION: This snippet demonstrates how to handle asynchronous operations and their cancellation in `AsyncNotifier`s, addressing the absence of a `mounted` property. Since `AsyncNotifier` instances are not disposed on rebuild, a `mounted` property would be misleading. Instead, `ref.onDispose` is used to cancel ongoing asynchronous tasks (e.g., network requests) using a `Completer` or similar mechanism, preventing `StateError`s if the notifier's state is rebuilt while an operation is pending.
SOURCE: https://github.com/rrousselgit/riverpod/blob/master/website/i18n/zh-Hans/docusaurus-plugin-content-docs/current/migration/from_state_notifier.mdx#_snippet_2

LANGUAGE: dart
CODE:
```
import 'dart:async';
import 'package:flutter_riverpod/flutter_riverpod.dart';

class MyAsyncNotifier extends AsyncNotifier<int> {
  @override
  Future<int> build() async {
    final cancelCompleter = Completer<void>();
    ref.onDispose(() {
      if (!cancelCompleter.isCompleted) {
        cancelCompleter.completeError(StateError('Operation cancelled'));
      }
    });

    try {
      // Simulate an async operation that can be cancelled
      await Future.delayed(const Duration(seconds: 2));
      if (cancelCompleter.isCompleted) {
        throw StateError('Operation was cancelled before completion');
      }
      return 42;
    } on StateError catch (e) {
      // Catch the cancellation error thrown by the completer
      print('Async operation caught cancellation: $e');
      return 0; // Or handle as appropriate
    } catch (e) {
      print('Other error during async operation: $e');
      rethrow;
    }
  }
}

final myAsyncNotifierProvider = AsyncNotifierProvider<MyAsyncNotifier, int>(MyAsyncNotifier.new);
```

    required this.description,
    this.completed = false,
  });

  final String id;
  final String description;
  final bool completed;

TITLE: Filtering a Todo List with ref.watch in Riverpod (Dart)
DESCRIPTION: This snippet demonstrates how to use `ref.watch` within a Riverpod provider to combine the states of multiple other providers. It creates a `filteredTodoListProvider` by watching `filterTypeProvider` and `todosProvider` to dynamically filter a list of tasks based on the current filter type, ensuring the filtered list automatically updates when either dependency changes.
SOURCE: https://github.com/rrousselgit/riverpod/blob/master/website/i18n/ko/docusaurus-plugin-content-docs/current/concepts/reading.mdx#_snippet_7

LANGUAGE: Dart
CODE:
```
enum FilterType { all, completed, uncompleted }

final filterTypeProvider = StateProvider<FilterType>((ref) => FilterType.all);

class Todo {
  Todo(this.id, this.description, this.completed);
  final String id;
  final String description;
  final bool completed;
}

final todosProvider = StateNotifierProvider<TodosNotifier, List<Todo>>((ref) {
  return TodosNotifier();
});

class TodosNotifier extends StateNotifier<List<Todo>> {
  TodosNotifier() : super([]);

  void addTodo(Todo todo) {
    state = [...state, todo];
  }

  void toggle(String id) {
    state = [
      for (final todo in state)
        if (todo.id == id) Todo(todo.id, todo.description, !todo.completed) else todo,
    ];
  }
}

final filteredTodoListProvider = Provider<List<Todo>>((ref) {
  final filter = ref.watch(filterTypeProvider);
  final todos = ref.watch(todosProvider);

  switch (filter) {
    case FilterType.completed:
      return todos.where((todo) => todo.completed).toList();
    case FilterType.uncompleted:
      return todos.where((todo) => !todo.completed).toList();
    case FilterType.all:
      return todos;
  }
});

TITLE: Fully Migrated Riverpod AsyncNotifier with Mutations (Dart)
DESCRIPTION: This snippet presents a fully migrated AsyncNotifier class, demonstrating how to handle both initialization and state mutations (side effects) efficiently. It highlights the removal of boilerplate code, such as manual 'isLoading'/'hasError' flags and extensive 'try'/'catch' blocks for state updates, by leveraging Riverpod's automatic AsyncValue management.
SOURCE: https://github.com/rrousselgit/riverpod/blob/master/website/i18n/zh-Hans/docusaurus-plugin-content-docs/current/migration/from_change_notifier.mdx#_snippet_2

LANGUAGE: dart
CODE:
```
import 'package:riverpod_annotation/riverpod_annotation.dart';

part 'todo_list_notifier.g.dart';

// Assume Todo class and _apiService are defined elsewhere.
// class Todo { ... }
// final _apiService = ApiService();

@riverpod
class TodoList extends _$TodoList {
  // The build method handles initial asynchronous data fetching.
  // Riverpod automatically manages the loading and error states of this Future.
  @override
  Future<List<Todo>> build() async {
    // This replaces manual isLoading/hasError flags for initial load.
    return _apiService.fetchTodos();
  }

  // Example mutation method: adding a new todo.
  // Riverpod simplifies state updates and error handling for mutations.
  Future<void> addTodo(String description) async {
    // Optionally set state to loading during mutation, or handle optimistically.
    // state = const AsyncValue.loading(); // If you want to show loading during mutation

    try {
      final newTodo = await _apiService.addTodo(description);
      // Update the state with the new list of todos.
      // Riverpod handles the AsyncValue.data wrapper.
      state = AsyncValue.data([...state.value!, newTodo]);
    } catch (e, st) {
      // If an error occurs during the mutation, update the state to an error state.
      // This replaces manual hasError flags and try/catch boilerplate for state management.
      state = AsyncValue.error(e, st);
    }
  }
}
